# Security Policy / 安全策略

## 中文

请不要在公开 Issue、讨论、PR、日志或审计报告中提交令牌、Cookie、恢复码、私有路径清单、账号信息或未脱敏配置。

报告安全问题时，请提供最小复现、受影响文件和预期边界，不要附带真实凭证。仓库启用 GitHub Private Vulnerability Reporting 后，优先使用该通道。

本项目中的 Skill 不应自动安装或卸载软件、删除文件或远端分支、强推或覆盖历史、创建公开仓库或改变可见性、上传私有文件或原始审计报告。

## English

Do not include tokens, cookies, recovery codes, private path inventories, account data, or unredacted configuration in public issues, discussions, pull requests, logs, or audit reports.

When reporting a security issue, provide a minimal reproduction, the affected file, and the expected boundary without real credentials. Prefer GitHub Private Vulnerability Reporting when it is enabled.

Skills in this project must not automatically install or uninstall software, delete files or remote branches, force-push, create public repositories or change repository visibility, or upload private files and raw reports.
